## 2. Performance + Security (rare and powerful)

Very few engineers combine:

```
security thinking
+
performance / systems thinking
```

Example problems:

- DoS vulnerabilities caused by algorithmic complexity
    
- memory exhaustion attacks
    
- CPU exhaustion from bad crypto or parsing
    
- event-loop blocking attacks in Node
    
- inefficient validation that attackers exploit
    

Example:

A login endpoint might be “secure” but can be abused to cause **CPU spikes** through password hashing abuse.

This area sits between:

```
AppSec
+
Performance engineering
```

And it’s very valuable because companies lose money from:

- outages
    
- slow systems
    
- security abuse
    

This combination fits you extremely well.

---

## 3. Runtime Security / Observability Security

This is a newer and very interesting area.

Instead of just scanning code, you **observe the running system** and detect threats.

Examples:

- detecting unusual syscalls
    
- detecting container escapes
    
- detecting suspicious process behavior
    
- runtime anomaly detection
    

Tools and concepts:

- eBPF
    
- syscall tracing
    
- container security
    
- runtime policy enforcement
    

This sits between:

```
Linux internals
+
security monitoring
+
performance tooling
```

Your embedded + Linux background would help a lot here.

This field is growing fast.
