---

# Why This Role Is Becoming Important

AI systems introduce three new categories of problems.

### 1. Security Problems

AI introduces attack surfaces that traditional security tools do not handle.

Examples:

- prompt injection
    
- model jailbreak
    
- data leakage
    
- model extraction
    
- training data poisoning
    

Security organizations like **OWASP** have already started publishing guidance for securing AI systems.

Someone must design **secure AI architectures**, not just scan code.

---

### 2. Performance Problems

AI inference is expensive.

Problems include:

- GPU underutilization
    
- slow inference latency
    
- inefficient batching
    
- high memory consumption
    

Tools used for monitoring these include:

- **Prometheus**
    
- **Grafana**
    
- **OpenTelemetry**
    

Companies constantly try to reduce inference cost because AI workloads burn huge compute budgets.

---

### 3. System Architecture Problems

Modern AI applications have many components:

```
API → model → embedding → vector DB → retrieval → response
```

Popular infrastructure pieces include:

- **Kubernetes**
    
- **Triton Inference Server**
    
- **Ray**
    
- **PyTorch**
    

These systems require **strong architecture thinking**.

---
