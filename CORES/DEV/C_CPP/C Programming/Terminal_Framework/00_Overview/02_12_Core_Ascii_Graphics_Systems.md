

Let’s walk through the **12 core techniques**.

---

# 1. Particle Systems

Almost every visual simulation starts with **particles**.

A particle is just:

```
struct Particle {
    vec2 position;
    vec2 velocity;
    float life;
};
```

Examples:

Matrix rain  
→ each character is a particle

Fireworks  
→ sparks are particles

Explosion  
→ debris particles

Fluid  
→ SPH particles

Particles are powerful because **complex visuals emerge from simple rules**.

---

# 2. Position + Velocity Integration

Movement is usually done with **Euler integration**.

Basic update:

```
velocity += acceleration * dt
position += velocity * dt
```

Example:

Fireworks

```
gravity = (0, -9.8)

velocity += gravity * dt
position += velocity * dt
```

This tiny formula drives **most motion in simulations**.

---

# 3. Forces

Forces modify velocity.

Common forces:

```
gravity
wind
drag
explosion impulse
pressure
spring force
```

Example explosion force:

```
force = normalize(pos - center) * strength
velocity += force
```

Many visual effects are just **clever force fields**.

---

# 4. Randomness (Stochastic Systems)

Randomness makes visuals **organic**.

Examples:

Matrix rain

```
random characters
random stream lengths
random spawn timing
```

Fireworks

```
random spread angles
random spark lifetime
```

Without randomness everything looks **too mechanical**.

---

# 5. Lifetime / Decay

Particles usually fade over time.

```
life -= dt

if life <= 0
    remove particle
```

This drives:

```
smoke fade
spark fade
raindrop disappearance
fluid evaporation
```

Decay makes systems **self-cleaning**.

---

# 6. Spawn / Emitters

Particles are usually generated by **emitters**.

Example:

Firework emitter

```
spawn 200 particles
random velocity directions
```

Matrix rain emitter

```
spawn column streams
```

Explosion emitter

```
spawn particles from center
```

Emitters control **where energy enters the system**.

---

# 7. Fields

Fields apply forces over space.

Example wind field:

```
velocity += wind(x,y)
```

Example explosion field:

```
force = normalize(pos - blast_center)
```

Fields create **global behavior**.

Used in:

```
fluids
smoke
explosions
gravity wells
```

---

# 8. Noise

Noise produces **natural variation**.

Instead of random jumps, noise is **smooth randomness**.

Example uses:

```
smoke movement
water waves
fire flicker
cloud motion
```

Common types:

```
Perlin noise
Simplex noise
value noise
```

Noise makes motion **flow instead of jitter**.

---

# 9. Grid / Spatial Structures

Many simulations store data in **grids**.

Example:

```
grid[x][y]
```

Used for:

```
fluid velocity
density fields
distance fields
terminal framebuffers
```

Your Matrix rain example uses a **grid framebuffer**.

Grids allow **fast spatial queries**.

---

# 10. Neighbor Interaction

Particles sometimes interact with nearby particles.

Example SPH fluid:

```
for neighbors within radius:
    compute pressure
```

Used for:

```
fluids
boids
crowds
soft bodies
```

Efficient neighbor search often uses:

```
spatial hash
uniform grid
kd-tree
```

---

# 11. Distance Fields

Distance fields are extremely powerful.

A function returns:

```
distance(point, object)
```

Example:

Sphere distance:

```
d = length(p - center) - radius
```

Used in:

```
raymarching
collision detection
soft shadows
metaballs
```

Distance fields power **modern procedural graphics**.

---

# 12. Iterative Rendering (Raymarching / Sampling)

Instead of drawing objects directly, some simulations **sample space repeatedly**.

Raymarching algorithm:

```
repeat:
    step along ray
    check distance field
```

This allows rendering:

```
3D shapes
fractals
volumetric fog
procedural worlds
```

Even in ASCII terminals.
